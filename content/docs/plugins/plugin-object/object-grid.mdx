A specialized grid component that automatically fetches and displays data from ObjectQL objects. Built on the data-table foundation and **fully aligned with @objectstack/spec view.zod ListView (grid type)** for maximum interoperability.

## Overview

ObjectGrid implements the grid view type from @objectstack/spec ListView schema:
- **Grid/Table Display**: CRUD operations with search, filters, and pagination
- **Inline Editing**: Spreadsheet-like editing with keyboard navigation (set `editable: true`)
- **View.zod Compliance**: Full compatibility with @objectstack/spec ListView grid type

Inherits all data-table features: sorting, filtering, pagination, row selection, CSV export, column resizing, and reordering.

> **✨ New in v0.3**: ObjectGrid (formerly ObjectTable) now aligns with @objectstack/spec view.zod grid type. See the [Migration Guide](/docs/plugins/plugin-object/object-grid-migration) for details.

## Traditional Mode - Basic Usage

```tsx
import { ObjectGrid } from '@object-ui/plugin-object';

function UsersTable() {
  return (
    <ObjectGrid 
      schema={{
        type: 'object-grid',
        objectName: 'users',
        fields: ['name', 'email', 'status', 'created_at'],
        operations: {
          create: true,
          update: true,
          delete: true
        },
        showSearch: true,
        showPagination: true,
        pageSize: 25
      }}
      dataSource={dataSource}
      onEdit={(record) => console.log('Edit:', record)}
      onDelete={(record) => console.log('Delete:', record)}
    />
  );
}
```

## Grid Mode - Inline Editing

Enable spreadsheet-like editing by setting `editable: true`:

```tsx
function InventoryGrid() {
  return (
    <ObjectGrid 
      schema={{
        type: 'object-grid',
        objectName: 'inventory',
        fields: ['sku', 'name', 'quantity', 'price', 'status'],
        // Grid mode features
        editable: true,              // Enable inline editing
        keyboardNavigation: true,    // Arrow keys, Tab, Enter
        frozenColumns: 2,            // Freeze first 2 columns
        resizableColumns: true
      }}
      dataSource={dataSource}
      onCellChange={(row, col, value) => {
        console.log(`Updated: Row ${row}, Column ${col}:`, value);
      }}
    />
  );
}
```

## Features from data-table

All ObjectGrid instances automatically include:

✅ **Multi-column sorting** - Click headers to sort (ascending/descending/none)  
✅ **Real-time search** - Search across all visible columns  
✅ **Pagination** - Navigate large datasets efficiently  
✅ **Row selection** - Single or multiple selection with callbacks  
✅ **CSV export** - Export filtered and sorted data  
✅ **Column resizing** - Drag column borders to resize  
✅ **Column reordering** - Drag column headers to reorder

## Field Type Rendering

ObjectGrid automatically uses specialized renderers based on field types:

- **Text**: text, textarea, markdown, html
- **Numeric**: number, currency, percent
- **Boolean**: checkmarks with color coding
- **Date/Time**: date, datetime, time with locale formatting
- **Selection**: select, multi-select with colored badges
- **Contact**: email (mailto links), phone (tel links), url (external links)
- **Files**: file, image with thumbnails
- **Relationships**: lookup, master-detail
- **Computed**: formula, summary (read-only)
- **User**: user, owner with avatars

## Schema Properties

### Common Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `type` | `'object-grid'` | Required | Component type |
| `objectName` | string | Required | Name of the ObjectQL object |
| `label` | string | - | Display label for the table (new) |
| `data` | [ViewData](#viewdata-schema) | - | Data provider configuration (new) |
| `columns` | string[] \| [ListColumn](#listcolumn-schema)[] | - | Column configuration (enhanced) |
| `selection` | [SelectionConfig](#selection-config) | - | Selection configuration (new) |
| `pagination` | [PaginationConfig](#pagination-config) | - | Pagination configuration (new) |
| `searchableFields` | string[] | - | Fields to enable for search (new) |
| `sort` | string \| [SortConfig](#sort-config)[] | - | Sort configuration (enhanced) |
| `filter` | any[] | - | Filter rules (new) |
| `resizable` | boolean | `true` | Enable column resizing |
| `striped` | boolean | `false` | Alternating row colors (new) |
| `bordered` | boolean | `false` | Show cell borders (new) |

### Legacy Properties (Still Supported)

For backward compatibility, these legacy properties are still supported:

| Property | Type | Replacement | Description |
|----------|------|-------------|-------------|
| `fields` | string[] | `columns` | Legacy field names array |
| `staticData` | any[] | `data: { provider: 'value', items: [...] }` | Legacy inline data |
| `selectable` | boolean \| 'single' \| 'multiple' | `selection.type` | Legacy selection mode |
| `pageSize` | number | `pagination.pageSize` | Legacy page size |
| `showSearch` | boolean | `searchableFields` | Legacy search toggle |
| `showPagination` | boolean | `pagination` presence | Legacy pagination toggle |
| `defaultSort` | object | `sort` | Legacy sort config |
| `defaultFilters` | object | `filter` | Legacy filter config |
| `resizableColumns` | boolean | `resizable` | Legacy resizing toggle |
| `title` | string | `label` | Legacy table caption |

See the [Migration Guide](/docs/plugins/plugin-object/object-grid-migration) for details on adopting new properties.

### ViewData Schema

Data provider configuration with three modes:

```tsx
// Object provider (ObjectStack protocol)
data: {
  provider: 'object',
  object: 'users'
}

// Value provider (inline/static data)
data: {
  provider: 'value',
  items: [{ name: 'Alice' }, { name: 'Bob' }]
}

// API provider (custom endpoints)
data: {
  provider: 'api',
  read: {
    url: '/api/users',
    method: 'GET',
    headers: { 'Authorization': 'Bearer token' }
  }
}
```

### ListColumn Schema

Enhanced column configuration:

```tsx
columns: [
  {
    field: 'name',         // Field name (required)
    label: 'Full Name',    // Display label
    width: 200,            // Column width in pixels
    align: 'left',         // 'left' | 'center' | 'right'
    sortable: true,        // Allow sorting
    resizable: true,       // Allow resizing
    wrap: false,           // Text wrapping
    type: 'text',          // Renderer type hint
    hidden: false          // Hide by default
  }
]
```

### Selection Config

Row selection configuration:

```tsx
selection: {
  type: 'none'      // or 'single' or 'multiple'
}
```

### Pagination Config

Pagination configuration:

```tsx
pagination: {
  pageSize: 25,
  pageSizeOptions: [10, 25, 50, 100]
}
```

### Sort Config

Sort configuration (array for multi-column):

```tsx
sort: [
  { field: 'name', order: 'asc' },
  { field: 'created_at', order: 'desc' }
]
```

### Grid Mode Properties

Add these properties to enable spreadsheet-like behavior:

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `editable` | boolean | `false` | Enable inline cell editing |
| `keyboardNavigation` | boolean | `true` | Arrow keys, Tab, Enter navigation |
| `frozenColumns` | number | `0` | Number of left-pinned columns |
| `resizableColumns` | boolean | `false` | Enable column resizing |

## Keyboard Shortcuts (Grid Mode)

When `editable` and `keyboardNavigation` are enabled:

| Shortcut | Action |
|----------|--------|
| `↑` `↓` `←` `→` | Navigate between cells |
| `Tab` | Move to next cell |
| `Shift + Tab` | Move to previous cell |
| `Enter` | Start editing selected cell |
| `Escape` | Cancel editing |
