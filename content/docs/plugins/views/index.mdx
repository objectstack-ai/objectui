Core Object UI views package providing smart components for CRUD operations with ObjectQL integration.

## Installation

```bash
npm install @object-ui/views @object-ui/data-objectql
```

## Overview

The Views package provides seamless integration with ObjectQL backends through smart components that automatically generate UI from ObjectQL object schemas. It's a core package for building data-driven applications with ObjectUI.

## Components

The plugin provides three main components for building CRUD interfaces:

### ObjectTable

Enterprise-grade table component with unified schema supporting both traditional CRUD and spreadsheet-like inline editing. Built on data-table foundation with automatic field type rendering.

**Features:**
- Auto-generated columns from ObjectQL schema
- 20+ specialized field renderers (currency, percent, date, select badges, etc.)
- Multi-column sorting, search, pagination, row selection
- CSV export, column resizing, column reordering
- Optional grid mode: inline editing, keyboard navigation, frozen columns (set `editable: true`)

**Single unified type:**
- `type: 'object-grid'` - Supports both traditional and grid modes

[Learn more about ObjectTable →](/docs/plugins/views/object-grid)

### ObjectForm
Smart forms with validation and schema integration. Supports both create and edit modes with automatic field generation.

[Learn more about ObjectForm →](/docs/plugins/views/object-form)

### ObjectView
Complete CRUD views combining list, detail, and edit modes in a single component. Ideal for building full-featured data management interfaces.

[Learn more about ObjectView →](/docs/plugins/views/object-view)

## Features

- **Schema-driven**: Automatically adapts to ObjectQL metadata
- **Type-aware rendering**: 20+ specialized field renderers (currency, percent, badges, avatars, etc.)
- **Enterprise table**: Built on data-table with sorting, search, pagination, CSV export
- **Dual modes**: Traditional CRUD or spreadsheet-like inline editing
- **Type-safe**: Full TypeScript support
- **Lazy-loaded**: Only loads when used
- **Permissions**: Applies field-level and object-level permissions
- **Validation**: Built-in validation according to schema rules

## Schema Integration

All Object components automatically integrate with ObjectQL's schema system to:

- **Field Types**: Display appropriate UI components based on field types (text, number, date, boolean, etc.)
- **Permissions**: Apply field-level and object-level permissions
- **Validation**: Validate data according to schema rules (required, format, range, etc.)
- **Relationships**: Handle relationships between objects (lookup, master-detail, etc.)
- **Computed Fields**: Display computed/formula fields

## Data Source Configuration

The Object plugin requires an ObjectQL data source to be configured:

```tsx
import { ObjectQLDataSource } from '@object-ui/data-objectql';

const dataSource = new ObjectQLDataSource({
  baseUrl: 'https://api.example.com',
  token: 'your-auth-token',
  // Optional configurations
  timeout: 30000,
  headers: {
    'X-Custom-Header': 'value'
  }
});
```

## TypeScript Support

```typescript
import type { 
  ObjectTableSchema,
  ObjectFormSchema,
  ObjectViewSchema 
} from '@object-ui/views';

// Object Table (Traditional CRUD)
const tableSchema: ObjectTableSchema = {
  type: 'object-grid',
  objectName: 'users',
  fields: ['name', 'email', 'role'],
  operations: {
    update: true,
    delete: true,
    export: true
  },
  showSearch: true,
  pageSize: 20
};

// Object Table (Grid Mode with inline editing)
const gridSchema: ObjectTableSchema = {
  type: 'object-grid',
  objectName: 'inventory',
  fields: ['sku', 'name', 'quantity', 'price'],
  editable: true,              // Enable grid mode
  keyboardNavigation: true,
  frozenColumns: 1
};

// Object Form
const formSchema: ObjectFormSchema = {
  type: 'object-form',
  objectName: 'contacts',
  mode: 'create',
  fields: ['first_name', 'last_name', 'email'],
  onSuccess: {
    type: 'toast',
    message: 'Contact created!'
  }
};

// Object View
const viewSchema: ObjectViewSchema = {
  type: 'object-view',
  objectName: 'products',
  view: 'list',
  listColumns: ['name', 'price', 'stock']
};
```
